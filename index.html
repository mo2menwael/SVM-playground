What is a Support Vector Machine?

A Support Vector Machine (SVM) is a supervised learning algorithm that finds the hyperplane that maximally separates two classes. The data points closest to the boundary are the support vectors — they are the only points that actually define the decision surface.

When data is not linearly separable, SVM uses the kernel trick to implicitly transform the input data into a higher-dimensional feature space where a linear separator can exist. Instead of computing the transformation directly, a kernel function measures similarities between data points efficiently. Common kernels include the linear, polynomial, and RBF (Gaussian) kernels, which allow SVMs to handle complex nonlinear classification problems.

Kernel types covered

Linear

K(x, y) = x · y

Best when data is (nearly) linearly separable. Fast and interpretable.

Polynomial

K(x, y) = (γ x · y + coef0)d

Captures curved boundaries. Degree d controls flexibility.

RBF (Gaussian)

K(x, y) = exp(−γ ‖x − y‖²)

The most popular kernel — works well for arbitrary, smooth boundaries.

Sigmoid

K(x, y) = tanh(γ x · y + coef0)

Behaves like a 2-layer neural net. Sensitive to parameters.

How to use

  1. Pick a class (1–10) and click on the canvas to scatter points, or pick a quick demo.
  2. Choose a kernel and adjust C, γ, degree, coef0.
  3. Click Train SVM for automatic training or you can click Step to see the training process.
  4. With 2 classes you get a binary SVM (boundary + margins shown). For 3–10 classes, multiclass classification can be handled using either one-vs-all or one-vs-one strategies, and the canvas is colored according to the predicted class regions.
  5. Use Export PNG to save the current visualization.

Multi-class strategies

SVM is fundamentally a binary classifier. To separate K classes (K ≥ 3) we need to combine multiple binary SVMs. There are two standard ways to do this:

One-vs-All (OvA)

Train K binary classifiers. Each one answers “is this point class i, yes or no?” The final prediction is the class whose classifier outputs the highest decision value (argmax).

Pros: Few classifiers (4 for K=4, 10 for K=10). Each classifier sees all the training data, which can help with small datasets.

Cons: Each binary problem is imbalanced — one class against K−1 others. SVM’s margin can suffer when classes are very unbalanced.

One-vs-One (OvO)

Train K(K−1)/2 binary classifiers, one for every pair of classes. Each classifier sees only the points from those two classes. The final prediction is the class with the most votes across all pairwise classifiers.

Pros: Each binary problem is balanced (similar number of points per class). Each problem is also smaller, so each individual SVM trains faster. This is what scikit-learn’s default SVC uses.

Cons: Many more classifiers as K grows (45 for K=10). Voting can produce ties.

Try both: on most datasets the predictions are nearly identical. The difference becomes visible on datasets where one class is much smaller than the others (OvA struggles, OvO usually doesn’t).

How to read the live stats

The stats panel shows three different progress signals. They tell you different stories — you only know the model is fully trained when all three agree.

Training points accuracy

Percentage of training points that fall on the correct side of the decision boundary. Saturates very early — for separable data it usually hits 100% on the very first iteration.

Hitting 100% accuracy does not mean the model is done! It just means a feasible separator exists. SVM keeps iterating to find the best separator, not just any feasible one.

Margin (1 / ‖w‖)

The width of the corridor between the two classes — the actual quantity SVM is maximizing. Computed in feature space as 1 / √( ΣΣ αiαj yiyj K(xi,xj) ).

Grows monotonically through training and asymptotes at the optimum. Watching it climb is the clearest sign the algorithm is still making progress, even when accuracy is already pinned at 100%.

KKT violations

Count of training points still violating the Karush–Kuhn–Tucker optimality conditions within tolerance. These are the points that still “want” to push the boundary.

Drops to 0 at convergence — this is the algorithm’s actual stopping signal. When it reads 0, the algorithm has found the max-margin separator and further training does nothing.

Bottom line: if you see accuracy = 100% but KKT violations > 0, the algorithm is still improving the margin — let it finish.
